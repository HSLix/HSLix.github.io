# 题目浅析
- 想查看原题可以点击[题目链接](https://leetcode.cn/problems/coin-change-ii/)。

- 简单地说，就是给一堆不同数额的硬币和一个目标数字，求能用硬币凑出目标数字的组合数。

# 思路分享
- 与 {% post_link 322零钱兑换 %} 思路相同，只不过最后统计的是组合数
> https://leetcode.cn/problems/coin-change-ii/solutions/2706227/shi-pin-wan-quan-bei-bao-cong-ji-yi-hua-o3ew0/

- 本题学到的一点在于要区分“组合”和“排列”，前者不区分答案中元素的前后，后者区分。那么在实际算法中如何区分呢？答案是遍历的先后顺序，如果先遍历物品，那么物品的选取顺序一定固定，不可能先选择大物又跳到小物，这就是排列；如果先遍历数额，那么物品的遍历顺序就随意了，也就是组合。

# 代码解答（强烈建议自行解答后再看）
- 参考题解
```python
class Solution:
    def change(self, amount: int, coins: List[int]) -> int:
        # 空间优化
        n = len(coins)
        f = [0]*(amount+1)
        f[0] = 1
        for i in range(n):
            for j in range(amount+1):
                if j >= coins[i]:
                    f[j] = f[j] + f[j-coins[i]]
                else:
                    f[j] = f[j]
        return f[amount]
        # 递推
        # f[i+1][j] = f[i][j] + f[i+1][j-coins[i]]
        n = len(coins)
        f = [[0]*(amount+1) for _ in range(n+1)]
        f[0][0] = 1
        for i in range(n):
            for j in range(amount+1):
                if j >= coins[i]:
                    f[i+1][j] = f[i][j] + f[i+1][j-coins[i]]
                else:
                    f[i+1][j] = f[i][j]
        # print(f)
        return f[n][amount]
        # 记忆化搜索
        sorted(coins)
        @cache
        def dfs(i:int, amount:int) -> int:
            if amount < 0:
                return 0
            if i == -1:
                return 1 if amount == 0 else 0
            return (dfs(i-1, amount) + dfs(i, amount-coins[i]))
        return dfs(len(coins)-1, amount)
```
