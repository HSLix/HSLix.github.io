【Git Error】error Invalid path 报错，clone 失败，文件夹空白问题解决

问题背景

可能是在 clone 某仓库后出现形如下面报错

而且仓库里除了.git文件夹就“一尘不染”。

解决方法

在仓库的文件夹中打开终端，并输入
git config core.protectNTFS false
大致意思是关闭文件保护之类的，然后切换为刚刚的分支，如
git checkout main
然后文件夹就正常咯
